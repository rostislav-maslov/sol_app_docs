# Epics

Корневая директория `epics/` используется в режиме `project-manager` для задач
и эпиков команды.

`ai-context/tasks/` при этом не заменяет `epics/`: в `ai-context/tasks/`
ведутся только задачи, которые должен выполнить сам AI-агент.

В этом source-of-truth репозитории `epics/` хранится как документационный
шаблон. При переносе в реальный проект эта директория должна копироваться
только в режиме `project-manager` и только как документация:

- `epics/README.md`
- `epics/_example/**/*`

Живой `epic-list.md` и реальные директории эпиков должны создаваться и вестись
уже в целевом проекте как project-local данные.

## Рекомендуемая структура

```text
epics/
  epic-list.md
  <код>/
    <код>-epic.md
    <код>.md
    context/
```

Где:

- `epic-list.md` - короткий список эпиков;
- `<код>/<код>-epic.md` - основная постановка эпика;
- `<код>/<код>.md` - декомпозиция на задачи для команды;
- `<код>/context/` - supporting-материалы.

## Правила

- одна директория = один эпик;
- код эпика должен быть стабильным и читаемым, например `EP-001`;
- `README.md` внутри директории конкретного эпика не используется для рабочих
  данных;
- в `epic-list.md` хранятся только короткие формулировки;
- вся развёрнутая детализация живет внутри директории конкретного эпика;
- если репозиторий используется как source-of-truth, живые рабочие эпики не
  должны затираться при обновлении baseline.

## Пример

В этом репозитории пример структуры лежит в [`_example`](./_example) и служит
только документацией.

В реальном рабочем репозитории живые эпики нужно хранить напрямую в
`epics/`, а не внутри `_example/`.
