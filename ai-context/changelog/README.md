# Changelog

Эта директория хранит отдельные append-only changelog-файлы об изменениях,
которые AI вносил в проект.

## Формат

- одно завершенное изменение = один отдельный changelog-файл;
- рекомендуемое имя файла: `YYYY-MM-DD_HHMMSS_short-title.md`;
- после создания changelog-файл больше не дописывается другими изменениями;
- каждый changelog-файл должен содержать:
  - время;
  - короткий заголовок;
  - список измененных файлов или областей;
  - краткое описание фактически выполненных изменений.

Отдельный файл на каждое изменение нужен в том числе затем, чтобы разные
изменения не конфликтовали между собой в git.

## Пример

```md
# Переход на one-change-one-changelog

- Время: `2026-03-24 15:47:59`
- Файлы: `ai-context/tasks/task-rules.md`, `ai-context/ai-rules/005_append-only-changelog.md`
- Что сделано: changelog переведен на модель «одно изменение = один файл».
```

## Исторические записи

Старые changelog-файлы, созданные по дневной схеме, можно сохранять без
массовой миграции.
